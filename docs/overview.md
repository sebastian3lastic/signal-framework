# Signal Framework --- Overview

Signal is a model-agnostic decision framework designed to evaluate
professional tools, resources, and emerging trends with structural
clarity.

It is not a chatbot.\
It is not a recommendation engine.\
It is not a productivity hack.

It is a protocol.

------------------------------------------------------------------------

## The Problem

In fast-moving domains (AI, design, technology, strategy), new tools and
ideas appear constantly.

Most professionals face recurring risks:

-   Reactive adoption (FOMO-driven decisions)
-   Tool stack fragmentation
-   Hidden opportunity cost behind "experimentation"

Signal converts trend exposure into structured decisions.

------------------------------------------------------------------------

## Core Principle

Separate judgment from impulse.

Instead of asking:

"Is this tool amazing?"

Signal asks:

"Does this structurally improve my direction?"

------------------------------------------------------------------------

## Architecture

Signal is composed of three layers:

### 1. Framework (Stable Engine)

The framework defines:

-   Evaluation criteria
-   Scoring scale
-   Risk inversion logic
-   Thresholds
-   Veto rules
-   Source validation policy
-   Output constraints

This layer is stable and versioned.

------------------------------------------------------------------------

### 2. Profile (Strategic Identity)

The profile defines:

-   Role
-   Short-term and long-term objectives
-   Operational principles
-   Optional threshold overrides
-   Risk tolerance (descriptive only)

Changing the profile adapts the framework without altering the engine.

------------------------------------------------------------------------

### 3. System Prompt (Execution Interpreter)

The system prompt instructs a Large Language Model to:

-   Load the YAML protocol
-   Execute scoring strictly according to defined criteria
-   Apply numeric thresholds exactly as written
-   Enforce veto rules
-   Format output consistently

The model is an executor.\
The YAML is the source of truth.

------------------------------------------------------------------------

## Model-Agnostic by Design

Signal is not tied to GPT, Gemini, Claude, or any specific provider.

Any LLM capable of:

-   Parsing YAML
-   Performing structured scoring
-   Following strict instructions

can execute the framework.

The protocol remains portable.

------------------------------------------------------------------------

## Decision Logic

Each resource is evaluated across defined criteria (standard and
inverse).

An average score is calculated.

Thresholds determine:

-   Integrate
-   Sandbox
-   Discard

Absolute veto rules override averages when structural risk is
unacceptable.

------------------------------------------------------------------------

## Versioning Philosophy

Signal follows semantic versioning principles:

-   MAJOR: Structural changes (criteria, scale, decision logic)
-   MINOR: Threshold adjustments or non-breaking enhancements
-   PATCH: Documentation and clarification updates

Protocol versions ensure decision reproducibility.

------------------------------------------------------------------------

## Intended Use

Signal is suited for professionals who:

-   Operate in high-change environments
-   Prefer structured adoption over reactive experimentation
-   Value reusable systems over temporary gains
-   Want versioned, reproducible decision logic
